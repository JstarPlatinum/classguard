# 空气质量综合评估报告与代码使用流程方案

> 版本：v1.0  
> 适用场景：教室、实验室、办公室、宿舍、小型公共室内空间  
> 适用硬件：ESP32-S3 + SCD41/同类 CO₂ 传感器 + PMS5003/同类颗粒物传感器 + 温湿度传感器 + 占有率检测模块  
> 生成日期：2026-05-10

---

## 1. 报告目标

本报告面向一个室内空气质量综合评估系统，核心目标不是只判断某一个指标是否超标，而是把 **CO₂、PM1.0、PM2.5、PM10、温度、湿度、占有率** 放入同一个评价框架中。这样做可以兼顾健康风险、学习/工作效率、热舒适性和空间使用负荷。

空气质量评价不宜简单使用平均值。举例来说，如果 CO₂ 已经达到 1800 ppm，但颗粒物和温湿度都正常，普通平均法可能仍给出“良好”结论；这种结果会掩盖通风不足带来的疲劳、注意力下降和室内代谢污染物积累问题。因此，本方案采用：

```text
综合加权评分 + 单项红线约束 + 占有率动态修正 + 提示文本生成
```

最终系统应同时输出：

- 综合空气质量得分
- 空气质量等级
- 主要影响因素
- 单项指标状态
- 触发的红线
- 面向用户的提示文本
- 面向设备控制的动作建议

## 2. 参考数据来源与依据

本方案中的阈值不是直接等同于某一个国家标准，而是综合了国际指南、室内空气质量标准、健康研究和实际嵌入式系统可用性后提出的工程化分级方案。

### 2.1 主要参考来源

| 来源 | 参考内容 | 用途 |
|---|---|---|
| WHO 2021 | PM2.5、PM10 指南 | 颗粒物分级 |
| EPA IAQ | 湿度 30%–50% | 湿度舒适区 |
| Harvard/PMC | CO₂ 与认知表现 | CO₂ 风险 |
| ASHRAE/Persily | CO₂ 不是单一 IAQ 指标 | CO₂ 解释边界 |
| GB/T 18883-2022 | 室内空气质量 | 国内参考 |

### 2.2 关键参考值

- WHO 2021 空气质量指南中，PM2.5 的 24 h 指南值为 15 μg/m³，年均值为 5 μg/m³；PM10 的 24 h 指南值为 45 μg/m³，年均值为 15 μg/m³。
- EPA 关于室内空气质量的建议中，室内相对湿度通常建议控制在 30%–50%。
- 多国办公建筑湿度研究指出，40%–60% RH 的中等湿度区间有利于降低部分病毒传播风险，并有助于减轻干燥刺激。
- CO₂ 在普通室内浓度下更常用作通风和人员代谢污染物积累的代理指标。受控实验显示，当 CO₂ 升高到约 950 ppm 与 1400 ppm 时，认知表现会出现不同程度下降。
- ASHRAE 相关讨论指出，不应把“1000 ppm CO₂”简单理解为当前标准中的固定室内 CO₂ 限值。CO₂ 判断需要结合空间类型、人员密度和通风目标。

## 3. 指标体系设计

本方案将空气质量拆分为四个维度：颗粒物健康风险、通风与人员代谢风险、热湿舒适风险、空间负荷风险。

### 3.1 指标分组

| 指标组 | 子指标 | 作用 |
|---|---|---|
| 颗粒物组 | PM1.0 | 细颗粒辅助判断 |
| 颗粒物组 | PM2.5 | 核心健康指标 |
| 颗粒物组 | PM10 | 扬尘与粗颗粒 |
| 通风组 | CO₂ | 通风代理指标 |
| 热舒适组 | 温度 | 热舒适 |
| 热舒适组 | 湿度 | 呼吸道舒适 |
| 负荷组 | 占有率 | 风险增长速度 |

PM1.0、PM2.5、PM10 之间存在包含关系。PM1.0 属于 PM2.5 的一部分，PM2.5 又属于 PM10 的一部分。因此，算法不宜把三者视为完全独立的污染物进行等权叠加。更合理的做法是让 PM2.5 成为颗粒物组的主指标，PM1.0 和 PM10 用于补充判断颗粒来源。

## 4. 综合权重方案

### 4.1 默认权重

| 指标 | 权重 | 定位 |
|---|---:|---|
| CO₂ | 30% | 通风核心 |
| PM2.5 | 22% | 健康核心 |
| PM10 | 10% | 粗颗粒 |
| PM1.0 | 8% | 细颗粒 |
| 温度 | 10% | 热舒适 |
| 湿度 | 10% | 湿舒适 |
| 占有率 | 10% | 负荷风险 |

也可以按指标组理解为：

```text
颗粒物组 40% + CO₂ 30% + 温湿度 20% + 占有率 10%
```

这个比例适用于教学、办公和比赛演示场景。它的特点是：CO₂ 足够突出，PM2.5 不会被弱化，温湿度能反映人体舒适度，占有率则用于预判空气质量恶化趋势。

### 4.2 动态权重修正

占有率升高时，CO₂ 与湿度的风险增长速度会明显加快。为了让系统具备提前预警能力，权重不应始终固定。

定义占有率：

\[
r = \frac{N_{current}}{N_{design}}
\]

其中，\(N_{current}\) 为当前人数，\(N_{design}\) 为设计容量。

动态修正建议如下：

| 条件 | 调整对象 | 调整方式 |
|---|---|---|
| r ≤ 0.8 | 全部权重 | 不调整 |
| 0.8 < r ≤ 1.0 | CO₂ | 适度增加 |
| 1.0 < r ≤ 1.2 | CO₂、湿度 | 同时增加 |
| r > 1.2 | CO₂、湿度、占有率 | 强化预警 |

推荐公式：

\[
W'_{CO2}=W_{CO2}\times \min(1.35, 1+0.6\times \max(0,r-0.8))
\]

\[
W'_{RH}=W_{RH}\times \min(1.20, 1+0.4\times \max(0,r-1.0))
\]

修正后需要重新归一化：

\[
\hat{W_i}=\frac{W'_i}{\sum W'_i}
\]

如果没有占有率数据，可以把占有率权重临时置为 0，并将其他指标权重归一化。这样系统不会因为一个辅助指标缺失而停止输出。

## 5. 单项评分算法

每个指标先映射为 0–100 分，再进入综合加权。单项得分越高，代表状态越好。

### 5.1 线性分段评分函数

为了避免阶跃式评分导致数值突变，建议使用线性插值。

若某指标浓度为 \(x\)，阈值点为：

\[
(x_1,s_1),(x_2,s_2),\ldots,(x_n,s_n)
\]

当 \(x_k \le x \le x_{k+1}\) 时：

\[
S(x)=s_k + \frac{x-x_k}{x_{k+1}-x_k}\times(s_{k+1}-s_k)
\]

当数值低于最低阈值时，得分取最高分；高于最高阈值时，得分取最低分。

## 6. CO₂ 评价规则

CO₂ 是判断通风不足和人员代谢污染物积累的重要代理指标。它不应被等同于全部空气质量，但在教室和办公场景中非常有价值。

### 6.1 CO₂ 分级

| CO₂ ppm | 得分 | 状态 | 提示文本 |
|---:|---:|---|---|
| ≤ 800 | 100 | 优秀 | 通风良好 |
| 800–1000 | 80 | 可接受 | 关注趋势 |
| 1000–1500 | 60 | 轻度风险 | 建议通风 |
| 1500–2000 | 40 | 明显风险 | 立即通风 |
| > 2000 | 20 | 严重风险 | 强制处理 |

### 6.2 CO₂ 红线

| 条件 | 总分上限 | 提示文本 | 动作建议 |
|---:|---:|---|---|
| > 1000 ppm | 75 | 通风不足 | 增加新风 |
| > 1500 ppm | 60 | 注意力风险 | 开窗/新风 |
| > 2000 ppm | 40 | 严重闷闭 | 疏散/强排 |
| > 3000 ppm | 30 | 高风险暴露 | 立即离开 |

### 6.3 CO₂ 用户提示模板

```text
当前 CO₂ 浓度偏高，说明室内通风不足或人员密度较大。建议立即开启新风、开窗通风，或减少室内人数。
```

如果室外颗粒物较高，不建议直接开窗。此时提示应改为：

```text
CO₂ 偏高，但室外颗粒物可能较高。建议优先使用带过滤的新风系统，避免直接开窗带入颗粒物。
```

## 7. 颗粒物评价规则

颗粒物组是健康风险判断的重点。PM2.5 对下呼吸道和心肺系统影响更明显，因此权重最高；PM10 更能体现扬尘和人员活动造成的再悬浮；PM1.0 用于补充判断细颗粒来源。

### 7.1 PM2.5 分级

| PM2.5 μg/m³ | 得分 | 状态 | 提示文本 |
|---:|---:|---|---|
| ≤ 15 | 100 | 优秀 | 颗粒物低 |
| 15–35 | 85 | 良好 | 继续监测 |
| 35–50 | 70 | 一般 | 建议净化 |
| 50–75 | 50 | 较差 | 开启净化 |
| 75–150 | 30 | 严重 | 强化净化 |
| > 150 | 15 | 危险 | 避免停留 |

### 7.2 PM10 分级

| PM10 μg/m³ | 得分 | 状态 | 提示文本 |
|---:|---:|---|---|
| ≤ 45 | 100 | 优秀 | 粗颗粒低 |
| 45–75 | 85 | 良好 | 继续监测 |
| 75–100 | 70 | 一般 | 减少扬尘 |
| 100–150 | 50 | 较差 | 清洁/净化 |
| 150–300 | 30 | 严重 | 控制扬尘 |
| > 300 | 15 | 危险 | 避免活动 |

### 7.3 PM1.0 分级

| PM1.0 μg/m³ | 得分 | 状态 | 提示文本 |
|---:|---:|---|---|
| ≤ 10 | 100 | 优秀 | 细颗粒低 |
| 10–20 | 85 | 良好 | 继续监测 |
| 20–35 | 70 | 一般 | 关注来源 |
| 35–50 | 50 | 较差 | 建议净化 |
| 50–100 | 30 | 严重 | 强化净化 |
| > 100 | 15 | 危险 | 避免暴露 |

PM1.0 缺乏统一的室内健康限值，因此本方案给出的 PM1.0 分级属于工程化建议值，适合用于相对风险判断。若后续有现场实测数据，应根据本地背景值和传感器特性重新校准。

### 7.4 颗粒物红线

| 条件 | 总分上限 | 提示文本 | 动作建议 |
|---:|---:|---|---|
| PM2.5 > 75 | 50 | 细颗粒严重 | 开启净化 |
| PM2.5 > 150 | 35 | 高风险颗粒 | 减少停留 |
| PM10 > 150 | 55 | 粗颗粒过高 | 控制扬尘 |
| PM10 > 300 | 40 | 扬尘严重 | 暂停活动 |
| PM1.0 > 50 | 60 | 细颗粒偏高 | 排查来源 |

### 7.5 颗粒物来源辅助判断

可以计算细颗粒比例：

\[
R_{fine}=\frac{PM1.0}{PM2.5}
\]

也可以计算粗颗粒比例：

\[
R_{coarse}=\frac{PM10-PM2.5}{PM10}
\]

判断建议：

| 条件 | 可能来源 | 处理建议 |
|---|---|---|
| Rfine > 0.7 | 烟雾/气溶胶 | 净化优先 |
| PM10 高 | 扬尘/走动 | 减少活动 |
| PM2.5 高 | 室外输入 | 过滤新风 |
| PM1.0 突升 | 燃烧/烟雾 | 排查污染源 |

## 8. 温湿度评价规则

温湿度不是传统意义上的污染物，但会显著影响人体舒适度、疲劳感、呼吸道状态以及霉菌生长风险。温度过高时，人会主观感到闷热；湿度过高时，霉菌和尘螨风险上升；湿度过低时，眼睛、皮肤和呼吸道更容易干燥。

### 8.1 温度分级

| 温度 °C | 得分 | 状态 | 提示文本 |
|---:|---:|---|---|
| 22–26 | 100 | 舒适 | 温度适宜 |
| 20–22/26–28 | 85 | 可接受 | 轻微偏离 |
| 18–20/28–30 | 65 | 预警 | 调节空调 |
| 16–18/30–32 | 45 | 较差 | 尽快调节 |
| <16 或 >32 | 25 | 严重 | 避免久留 |

### 8.2 湿度分级

| 湿度 RH | 得分 | 状态 | 提示文本 |
|---:|---:|---|---|
| 40%–60% | 100 | 舒适 | 湿度适宜 |
| 30%–40%/60%–70% | 85 | 可接受 | 轻微偏离 |
| 25%–30%/70%–80% | 65 | 预警 | 调节湿度 |
| 20%–25%/80%–85% | 45 | 较差 | 尽快处理 |
| <20% 或 >85% | 25 | 严重 | 避免久留 |

### 8.3 温湿度红线

| 条件 | 总分上限 | 提示文本 | 动作建议 |
|---:|---:|---|---|
| 温度 > 30°C | 70 | 偏热 | 降温通风 |
| 温度 > 32°C | 60 | 高温不适 | 立即降温 |
| 湿度 > 80% | 65 | 潮湿明显 | 除湿通风 |
| 湿度 > 85% | 60 | 霉菌风险 | 强制除湿 |
| 湿度 < 25% | 70 | 空气干燥 | 适度加湿 |
| 湿度 < 20% | 60 | 严重干燥 | 加湿补水 |

## 9. 占有率评价规则

占有率不代表污染浓度，但可以预测风险变化速度。人数越多，CO₂ 和湿度上升越快，颗粒物也可能因走动、衣物摩擦和地面再悬浮而增加。

### 9.1 占有率分级

| 占有率 r | 得分 | 状态 | 提示文本 |
|---:|---:|---|---|
| ≤ 0.6 | 100 | 宽松 | 人员较少 |
| 0.6–0.8 | 80 | 正常 | 保持监测 |
| 0.8–1.0 | 60 | 接近满载 | 提前通风 |
| 1.0–1.2 | 40 | 超负荷 | 减少人员 |
| > 1.2 | 20 | 拥挤 | 建议分流 |

### 9.2 占有率红线

| 条件 | 总分上限 | 提示文本 | 动作建议 |
|---:|---:|---|---|
| r > 1.0 | 70 | 人员超载 | 增加新风 |
| r > 1.2 | 60 | 明显拥挤 | 人员分流 |
| r > 1.5 | 50 | 高负荷空间 | 控制进入 |

## 10. 综合得分与等级

### 10.1 综合得分

设每个指标的单项得分为 \(S_i\)，归一化后的权重为 \(W_i\)，则综合加权得分为：

\[
S_{weighted}=\sum_i W_iS_i
\]

红线约束后的显示得分为：

\[
S_{display}=\min(S_{weighted},S_{cap})
\]

其中 \(S_{cap}\) 是所有触发红线中的最低总分上限。

### 10.2 综合等级

| 得分 | 等级 | 显示文本 |
|---:|---|---|
| 85–100 | 优秀 | 空气质量优秀 |
| 70–85 | 良好 | 空气质量良好 |
| 55–70 | 一般 | 空气质量一般 |
| 40–55 | 较差 | 空气质量较差 |
| < 40 | 严重 | 空气质量严重 |

### 10.3 总体提示文本规则

系统应优先展示对人体影响最大的因素。建议按以下优先级生成提示：

```text
红线指标 > 最低单项得分 > 权重最高异常项 > 趋势快速上升项
```

示例：

```text
综合评分 58，空气质量一般。主要问题是 CO₂ 偏高，说明通风不足。建议开启新风或开窗通风，并继续观察 PM2.5 是否上升。
```

## 11. 多场景综合研判

### 11.1 CO₂ 高、PM 低

这种情况通常说明室内人员密集、通风不足，但颗粒物污染并不明显。系统应优先建议通风。如果室外空气质量良好，可以开窗；如果室外 PM2.5 较高，则应使用带过滤的新风系统。

推荐提示：

```text
CO₂ 偏高，颗粒物正常。当前主要问题是通风不足，建议增加新风量或减少室内人数。
```

### 11.2 PM 高、CO₂ 低

这种情况说明污染源可能来自室外输入、扬尘、清扫、打印、燃烧或人员活动。此时盲目开窗可能让颗粒物进一步升高。

推荐提示：

```text
颗粒物偏高，但 CO₂ 正常。建议关闭污染源并开启空气净化，不建议直接开窗。
```

### 11.3 CO₂ 与 PM 同时高

这是较差场景，说明通风不足和颗粒物积累同时存在。只开净化器无法降低 CO₂，只开窗又可能带入颗粒物。因此应优先采用过滤新风与空气净化联动。

推荐提示：

```text
CO₂ 与颗粒物均偏高。建议同时开启过滤新风和空气净化，并减少室内人员停留时间。
```

### 11.4 温度高、湿度高

这类环境容易造成闷热、疲劳和霉菌风险。即使污染物浓度尚可，用户体验也会明显下降。

推荐提示：

```text
当前温湿度偏高，人体容易感到闷热。建议降温除湿，并适度增加通风。
```

### 11.5 占有率高、CO₂ 暂时正常

这是预警场景。系统不应等 CO₂ 超标后才提示，而应提前提醒用户通风。

推荐提示：

```text
当前人员密度较高，CO₂ 可能快速上升。建议提前开启新风系统。
```

## 12. 代码使用流程方案

该算法建议按“PC 端验证 → 嵌入式移植 → 前端展示 → 联合调试”的顺序开发。这样可以降低调试难度，避免一开始就在 ESP32-S3 上处理大量逻辑。

### 12.1 推荐工程结构


### 12.2 数据输入格式

PC 端和 ESP32 端都建议统一使用以下字段：

```json
{
  "co2_ppm": 980,
  "pm1_ugm3": 8.5,
  "pm25_ugm3": 18.0,
  "pm10_ugm3": 42.0,
  "temperature_c": 26.8,
  "humidity_rh": 58.0,
  "occupancy_count": 16,
  "design_capacity": 20,
  "outdoor_pm25_ugm3": 35.0
}
```

其中 `outdoor_pm25_ugm3` 是可选字段。如果没有室外 PM 数据，系统仍然可以输出评分，但无法精确判断“是否适合开窗”。

### 12.3 数据预处理流程

传感器数据会受到瞬时噪声、串口丢包和异常读数影响。正式评分前应进行预处理：

```text
读取传感器 → 单位统一 → 合理性检查 → 滑动中值滤波 → EMA 平滑 → 算法评分 → 输出结果
```

推荐参数：

| 项目 | 建议值 | 说明 |
|---|---:|---|
| 采样周期 | 1–5 s | 室内监测 |
| 上传周期 | 5–30 s | Web 展示 |
| 中值窗口 | 3–5 | 抑制突刺 |
| EMA α | 0.2–0.4 | 平滑曲线 |
| 报警延时 | 2–3 次 | 防误报 |

### 12.4 Python 核心算法代码

下面代码可直接作为 PC 端验证版本。后续移植到 ESP32-S3 时，可以保留同样的阈值和逻辑，只需改写为 C/C++ 函数。

```python
from dataclasses import dataclass
from typing import Dict, Any, List, Optional, Tuple


BASE_WEIGHTS = {
    "co2": 0.30,
    "pm25": 0.22,
    "pm10": 0.10,
    "pm1": 0.08,
    "temperature": 0.10,
    "humidity": 0.10,
    "occupancy": 0.10,
}


@dataclass
class AirQualityResult:
    weighted_score: float
    display_score: float
    level: str
    scores: Dict[str, float]
    weights: Dict[str, float]
    redlines: List[str]
    main_reasons: List[str]
    user_message: str
    action: str


def clamp(value: float, low: float, high: float) -> float:
    return max(low, min(high, value))


def piecewise_score(x: float, points: List[Tuple[float, float]]) -> float:
    """
    points: [(threshold, score), ...]
    阈值从小到大排列。
    """
    if x <= points[0][0]:
        return points[0][1]
    if x >= points[-1][0]:
        return points[-1][1]

    for i in range(len(points) - 1):
        x0, y0 = points[i]
        x1, y1 = points[i + 1]
        if x0 <= x <= x1:
            ratio = (x - x0) / (x1 - x0)
            return y0 + ratio * (y1 - y0)
    return points[-1][1]


def comfort_score_interval(x: float,
                           ideal_low: float,
                           ideal_high: float,
                           accept_low: float,
                           accept_high: float,
                           warn_low: float,
                           warn_high: float,
                           bad_low: float,
                           bad_high: float) -> float:
    if ideal_low <= x <= ideal_high:
        return 100.0
    if accept_low <= x < ideal_low:
        return piecewise_score(x, [(accept_low, 85), (ideal_low, 100)])
    if ideal_high < x <= accept_high:
        return piecewise_score(x, [(ideal_high, 100), (accept_high, 85)])
    if warn_low <= x < accept_low:
        return piecewise_score(x, [(warn_low, 65), (accept_low, 85)])
    if accept_high < x <= warn_high:
        return piecewise_score(x, [(accept_high, 85), (warn_high, 65)])
    if bad_low <= x < warn_low:
        return piecewise_score(x, [(bad_low, 45), (warn_low, 65)])
    if warn_high < x <= bad_high:
        return piecewise_score(x, [(warn_high, 65), (bad_high, 45)])
    return 25.0


def get_level(score: float) -> str:
    if score >= 85:
        return "优秀"
    if score >= 70:
        return "良好"
    if score >= 55:
        return "一般"
    if score >= 40:
        return "较差"
    return "严重"


def normalize_weights(weights: Dict[str, float], active_keys: List[str]) -> Dict[str, float]:
    active_weights = {k: v for k, v in weights.items() if k in active_keys and v > 0}
    total = sum(active_weights.values())
    if total <= 0:
        raise ValueError("No active weights available.")
    return {k: v / total for k, v in active_weights.items()}


def evaluate_air_quality(data: Dict[str, Any]) -> AirQualityResult:
    co2 = data.get("co2_ppm")
    pm1 = data.get("pm1_ugm3")
    pm25 = data.get("pm25_ugm3")
    pm10 = data.get("pm10_ugm3")
    temp = data.get("temperature_c")
    rh = data.get("humidity_rh")
    occ = data.get("occupancy_count")
    cap = data.get("design_capacity")
    outdoor_pm25 = data.get("outdoor_pm25_ugm3")

    scores: Dict[str, float] = {}
    redlines: List[str] = []
    cap_score = 100.0

    if co2 is not None:
        scores["co2"] = piecewise_score(
            float(co2),
            [(400, 100), (800, 100), (1000, 80), (1500, 60), (2000, 40), (3000, 20), (5000, 5)]
        )
        if co2 > 3000:
            cap_score = min(cap_score, 30)
            redlines.append("CO₂ > 3000 ppm")
        elif co2 > 2000:
            cap_score = min(cap_score, 40)
            redlines.append("CO₂ > 2000 ppm")
        elif co2 > 1500:
            cap_score = min(cap_score, 60)
            redlines.append("CO₂ > 1500 ppm")
        elif co2 > 1000:
            cap_score = min(cap_score, 75)
            redlines.append("CO₂ > 1000 ppm")

    if pm25 is not None:
        scores["pm25"] = piecewise_score(
            float(pm25),
            [(0, 100), (15, 100), (35, 85), (50, 70), (75, 50), (150, 30), (250, 15)]
        )
        if pm25 > 150:
            cap_score = min(cap_score, 35)
            redlines.append("PM2.5 > 150 μg/m³")
        elif pm25 > 75:
            cap_score = min(cap_score, 50)
            redlines.append("PM2.5 > 75 μg/m³")

    if pm10 is not None:
        scores["pm10"] = piecewise_score(
            float(pm10),
            [(0, 100), (45, 100), (75, 85), (100, 70), (150, 50), (300, 30), (500, 15)]
        )
        if pm10 > 300:
            cap_score = min(cap_score, 40)
            redlines.append("PM10 > 300 μg/m³")
        elif pm10 > 150:
            cap_score = min(cap_score, 55)
            redlines.append("PM10 > 150 μg/m³")

    if pm1 is not None:
        scores["pm1"] = piecewise_score(
            float(pm1),
            [(0, 100), (10, 100), (20, 85), (35, 70), (50, 50), (100, 30), (150, 15)]
        )
        if pm1 > 50:
            cap_score = min(cap_score, 60)
            redlines.append("PM1.0 > 50 μg/m³")

    if temp is not None:
        scores["temperature"] = comfort_score_interval(
            float(temp),
            ideal_low=22, ideal_high=26,
            accept_low=20, accept_high=28,
            warn_low=18, warn_high=30,
            bad_low=16, bad_high=32,
        )
        if temp > 32:
            cap_score = min(cap_score, 60)
            redlines.append("温度 > 32°C")
        elif temp > 30:
            cap_score = min(cap_score, 70)
            redlines.append("温度 > 30°C")

    if rh is not None:
        scores["humidity"] = comfort_score_interval(
            float(rh),
            ideal_low=40, ideal_high=60,
            accept_low=30, accept_high=70,
            warn_low=25, warn_high=80,
            bad_low=20, bad_high=85,
        )
        if rh > 85:
            cap_score = min(cap_score, 60)
            redlines.append("湿度 > 85%RH")
        elif rh > 80:
            cap_score = min(cap_score, 65)
            redlines.append("湿度 > 80%RH")
        elif rh < 20:
            cap_score = min(cap_score, 60)
            redlines.append("湿度 < 20%RH")
        elif rh < 25:
            cap_score = min(cap_score, 70)
            redlines.append("湿度 < 25%RH")

    occupancy_ratio: Optional[float] = None
    if occ is not None and cap is not None and cap > 0:
        occupancy_ratio = float(occ) / float(cap)
        scores["occupancy"] = piecewise_score(
            occupancy_ratio,
            [(0, 100), (0.6, 100), (0.8, 80), (1.0, 60), (1.2, 40), (1.5, 20), (2.0, 10)]
        )
        if occupancy_ratio > 1.5:
            cap_score = min(cap_score, 50)
            redlines.append("占有率 > 150%")
        elif occupancy_ratio > 1.2:
            cap_score = min(cap_score, 60)
            redlines.append("占有率 > 120%")
        elif occupancy_ratio > 1.0:
            cap_score = min(cap_score, 70)
            redlines.append("占有率 > 100%")

    weights = BASE_WEIGHTS.copy()
    if occupancy_ratio is not None:
        if occupancy_ratio > 0.8:
            weights["co2"] *= min(1.35, 1 + 0.6 * (occupancy_ratio - 0.8))
        if occupancy_ratio > 1.0:
            weights["humidity"] *= min(1.20, 1 + 0.4 * (occupancy_ratio - 1.0))

    active_keys = list(scores.keys())
    weights = normalize_weights(weights, active_keys)

    weighted_score = sum(scores[k] * weights[k] for k in active_keys)
    display_score = min(weighted_score, cap_score)
    level = get_level(display_score)

    sorted_reasons = sorted(scores.items(), key=lambda kv: kv[1])
    main_reasons = [name for name, value in sorted_reasons[:3] if value < 75]

    user_message, action = build_message(
        level=level,
        display_score=display_score,
        main_reasons=main_reasons,
        redlines=redlines,
        co2=co2,
        pm25=pm25,
        temp=temp,
        rh=rh,
        outdoor_pm25=outdoor_pm25,
    )

    return AirQualityResult(
        weighted_score=round(weighted_score, 2),
        display_score=round(display_score, 2),
        level=level,
        scores={k: round(v, 2) for k, v in scores.items()},
        weights={k: round(v, 4) for k, v in weights.items()},
        redlines=redlines,
        main_reasons=main_reasons,
        user_message=user_message,
        action=action,
    )


def build_message(level: str,
                  display_score: float,
                  main_reasons: List[str],
                  redlines: List[str],
                  co2: Optional[float],
                  pm25: Optional[float],
                  temp: Optional[float],
                  rh: Optional[float],
                  outdoor_pm25: Optional[float]) -> Tuple[str, str]:
    reason_map = {
        "co2": "CO₂ 偏高",
        "pm25": "PM2.5 偏高",
        "pm10": "PM10 偏高",
        "pm1": "PM1.0 偏高",
        "temperature": "温度不适",
        "humidity": "湿度不适",
        "occupancy": "人员密度较高",
    }
    reasons = [reason_map.get(r, r) for r in main_reasons]

    if redlines:
        prefix = f"综合评分 {display_score:.0f}，空气质量{level}。已触发红线：{'、'.join(redlines)}。"
    else:
        prefix = f"综合评分 {display_score:.0f}，空气质量{level}。"

    if co2 is not None and co2 > 1500:
        if outdoor_pm25 is not None and outdoor_pm25 > 50:
            return prefix + "当前主要问题是通风不足，但室外颗粒物偏高。建议开启过滤新风或空气净化联动。", "filtered_ventilation"
        return prefix + "当前主要问题是 CO₂ 偏高。建议立即开窗或开启新风，并适当减少人员密度。", "ventilation"

    if pm25 is not None and pm25 > 75:
        return prefix + "当前主要问题是颗粒物偏高。建议开启空气净化器，关闭污染源，减少人员活动。", "purify_air"

    if temp is not None and temp > 30:
        return prefix + "当前温度偏高，人体容易感到闷热。建议降温并配合通风。", "cooling"

    if rh is not None and rh > 80:
        return prefix + "当前湿度偏高，存在潮湿和霉菌风险。建议除湿并检查水汽来源。", "dehumidify"

    if reasons:
        return prefix + f"主要影响因素为：{'、'.join(reasons)}。建议根据异常项进行处理。", "improve_by_reason"

    return prefix + "各项指标整体稳定，可以维持当前运行状态。", "keep_monitoring"


if __name__ == "__main__":
    sample = {
```


### 13.3 ESP32 端伪代码

```cpp
void loop() {
    SensorData raw = readAllSensors();

    if (!isValid(raw)) {
        showError("传感器异常");
        delay(2000);
        return;
    }

    SensorData filtered = filterSensorData(raw);
    AirQualityResult result = evaluateAirQuality(filtered);

    showOnDisplay(result);
    uploadJson(result);
    controlDeviceByAction(result.action);

    delay(5000);
}
```

## 16. 参数校准建议

算法初始阈值用于原型开发。实际部署后，应至少收集 3–7 天数据，用于调整本地背景值。

### 16.1 建议记录字段

```text
时间、CO₂、PM1.0、PM2.5、PM10、温度、湿度、占有率、综合得分、用户反馈
```

### 16.2 校准思路

如果某一场景长期 PM2.5 背景值较高，但用户不希望系统频繁报警，可以调整“提示阈值”，但不建议随意提高“严重红线”。红线代表健康风险边界，不应仅根据主观体验放宽。

如果 CO₂ 经常快速升高，说明通风设计不足。此时不建议只降低 CO₂ 权重，而应保留算法敏感性，把它作为系统改进依据。

## 17. 最终建议

本方案适合用于室内空气质量监测原型、ESP32-S3 智能环境监测设备、教室/实验室环境管理系统以及创客竞赛项目展示。实际系统中，建议将综合得分作为用户入口，把单项指标和红线提示作为解释依据。这样的设计既能让普通用户快速理解空气质量，也能让评委或技术人员看清算法逻辑。

推荐最终显示格式如下：

```text
综合评分：76 / 良好
主要影响因素：CO₂ 略高、占有率偏高
当前建议：保持通风，继续观察 CO₂ 趋势
红线状态：未触发
系统动作：保持监测
```

从工程角度看，空气质量评价系统的价值不在于单纯显示传感器数值，而在于把多源数据转化为可解释、可执行的环境管理建议。通过加权评分、动态修正、红线约束和提示文本生成，本方案能够较好地支持空气质量的综合研判。
